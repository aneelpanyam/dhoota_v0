# Step 12: File Storage

## Goal

Integrate S3 for file uploads (activity attachments, file-type question answers) with CloudFront CDN for fast delivery, completing the file handling story.

## Acceptance Criteria

- [ ] S3 integration (`src/lib/s3/index.ts`):
  - `generateUploadUrl(key, contentType, maxSize)` — returns presigned PUT URL
  - `generateDownloadUrl(key)` — returns presigned GET URL (or CloudFront URL)
  - `deleteFile(key)` — deletes file from S3
- [ ] S3 key structure: `{spaceId}/{categoryId}/{activityId}/{filename}` — organized by ownership
- [ ] Upload configuration: max file size (configurable, default 10MB), allowed content types (images, documents, common file types)
- [ ] File upload component (`src/components/ui/file-upload.tsx`):
  - Drag-and-drop zone (desktop) + tap to select (mobile)
  - Upload progress indicator
  - Preview for images, file icon for other types
  - Remove/replace functionality
- [ ] Activity creation integration: file upload works in the creation wizard (Step 06 placeholder replaced)
  - Attachments field: upload files, stored as references in `activities.attachments` JSONB
  - File question type: upload files as answer to file-type questions, stored in `activity_answers.value`
- [ ] Activity feed/detail: attachments rendered (image thumbnails, file download links)
- [ ] CloudFront CDN:
  - Configuration documented (origin: S3 bucket, behavior: presigned URLs or signed cookies)
  - Download URLs use CloudFront domain when configured, fallback to direct S3 presigned URLs
- [ ] Upload validation: server-side content type and size validation before generating presigned URL
- [ ] All upload/download operations logged with trace_id

## Notes

- Presigned URLs are the pattern: the client uploads directly to S3 using a presigned PUT URL generated by the server. This avoids routing large files through the Next.js server.
- S3 configuration: bucket name, region, access key, secret key should be in environment variables. Document in `.env.example`.
- CloudFront setup is infrastructure — document the required configuration but don't try to create AWS resources programmatically. The app code should work with or without CloudFront (fallback to S3 presigned URLs).
- Image previews: for uploaded images, generate a thumbnail URL (can be same S3 URL with query params if using S3 image processing, or just use the full image with CSS sizing).
- File upload component should handle multiple files if needed (activity can have multiple attachments).
- Consider upload error recovery: if upload to S3 succeeds but saving the reference to DB fails, the orphan file exists. Document this as a known trade-off (cleanup job for later).
